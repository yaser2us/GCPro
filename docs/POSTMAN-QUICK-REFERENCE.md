# Postman Collection Quick Reference Card

Keep this open while creating Postman collections.

---

## 🎯 The Golden Rule

**THE CODE IS THE TRUTH. FOLLOW IT EXACTLY.**

1. DDL → Entity → DTO → Controller
2. Never invent field names
3. Never assume schema
4. Always verify before creating

---

## 📋 4-Step Verification Process

### 1️⃣ Check DDL
```bash
# Location
docs/database/FULL-DDL.md

# What to find
- Table name
- Column names (EXACT)
- Required vs optional (NULL/NOT NULL)
- Default values
```

### 2️⃣ Check Entity
```bash
# Location
src/plugins/{plugin}/entities/{entity}.entity.ts

# What to find
@Column({ type: 'varchar', length: 255 })
field_name: string;  // ← Use this exact name

# Optional fields have:
nullable: true
```

### 3️⃣ Check DTO
```bash
# Location
src/plugins/{plugin}/dto/{command}.request.dto.ts

# What to find
@IsOptional()  // ← Field is optional
@IsString()
field_name: string;  // ← Use this exact name
```

### 4️⃣ Check Controller
```bash
# Location
src/plugins/{plugin}/controllers/{plugin}.controller.ts

# What to find
@Post('v1/entities')  // ← HTTP method + route
@Headers('idempotency-key')  // ← Required headers
async createEntity(@Body() request: EntityCreateRequestDto)
```

## 🚫 Common Mistakes

| ❌ WRONG | ✅ CORRECT | Why? |
|---------|-----------|------|
| `firstName` | `full_name` | Check entity file |
| `date_of_birth` | `dob` | Check DDL |
| `email` in Person | `email` in User | Wrong table |
| `identity_type` | `id_type` | Check DDL |
| `POST /entities/123` | `GET /entities/123` | Check controller |
| No headers | X-User-Id + Idempotency-Key | Check controller |
| `POST /v1/person-identities/:id/verify` | **Route doesn't exist!** | Check controller - don't invent routes |

## 🔧 Required Postman Setup

### Every Collection Needs:

#### 1. Pre-Request Script
```javascript
// Add to collection-level pre-request script
pm.request.headers.add({
  key: 'X-User-Id',
  value: pm.collectionVariables.get('user_id') || '1'
});

if (pm.request.method === 'POST' || pm.request.method === 'PUT') {
  const existingKey = pm.request.headers.get('Idempotency-Key');
  if (!existingKey) {
    pm.request.headers.add({
      key: 'Idempotency-Key',
      value: pm.variables.replaceIn('{{$guid}}')
    });
  }
}
```

#### 2. Collection Variables
```json
{
  "variable": [
    {"key": "base_url", "value": "http://localhost:3000"},
    {"key": "user_id", "value": "1"}
  ]
}
```

#### 3. Test Script (Save IDs)
```javascript
// Add to individual request test script
var jsonData = pm.response.json();
pm.environment.set("entity_id", jsonData.entity_id);
```

## 🎯 Request Template

### POST Request Example
```
Method: POST
URL: {{base_url}}/v1/entities

Headers (auto-added by pre-request):
  Content-Type: application/json
  X-User-Id: 1
  Idempotency-Key: <auto-generated>

Body (from DTO):
{
  "required_field": "value",    // No @IsOptional
  "optional_field": "value"     // Has @IsOptional
}
```

### GET Request Example
```
Method: GET
URL: {{base_url}}/v1/entities/{{entity_id}}

Headers (auto-added):
  X-User-Id: 1

Body: (none)
```

## 🔍 Quick Diagnostic Commands

### Check what DTO expects
```bash
cat src/plugins/{plugin}/dto/{command}.request.dto.ts
```

### Check entity fields
```bash
cat src/plugins/{plugin}/entities/{entity}.entity.ts | grep "@Column"
```

### Check controller routes
```bash
grep -A 5 "@Post\|@Get\|@Put\|@Delete" \
  src/plugins/{plugin}/controllers/{plugin}.controller.ts
```

### Check DDL for table
```bash
grep -A 30 "CREATE TABLE \`{table}\`" docs/database/FULL-DDL.md
```

## ⚡ Quick Decision Tree

```
Creating new Postman request?
│
├─→ Step 1: Read DTO file
│   └─→ Use EXACT property names from DTO
│
├─→ Step 2: Check Entity file
│   └─→ Verify property names match

├─→ Step 3: Check Controller
```

## 🐛 Error Message → Fix

| Error Message | Fix |
|--------------|-----|
| `Missing X-User-Id header` | Add pre-request script |
| `Idempotency-Key header is required` | Add pre-request script for POST/PUT |
| `Unknown column 'field_name'` | Field doesn't exist, check DDL |
| `field_name is required` | Add required field from DTO |
| `User lacks required permissions` | Run bootstrap script |
| `NaN` in SQL | Wrong route/param, check controller |
| `TYPE_REQUIRED` | Missing required field from DTO |
| `404 Not Found` | Wrong URL, check controller route |
| `Cannot POST /v1/path` | Route doesn't exist - check controller for actual routes |

## ✅ Pre-Flight Checklist

Before creating request:
- [ ] Read DTO file
- [ ] Read Entity file
- [ ] Read Controller file
- [ ] Verified exact field names
- [ ] No invented fields
- [ ] No fields from other tables

Before testing request:
- [ ] Pre-request script added
- [ ] Collection variables set
- [ ] Bootstrap script run (user_id=1 exists)
- [ ] App is running

Before saving collection:
- [ ] All requests tested
- [ ] All errors fixed
- [ ] Test scripts save IDs
- [ ] Saved to `postman/{plugin}-api.postman_collection.json`

## 💡 Pro Tips

1. **Copy-paste field names** from DTO - don't type them manually
2. **Use VSCode search** to find entity/DTO files quickly
3. **Test as you go** - don't create all requests then test
4. **Check git diff** before committing to catch typos
5. **When in doubt, read the code** - it never lies
